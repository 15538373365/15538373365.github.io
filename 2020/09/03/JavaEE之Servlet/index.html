JavaEE之Servlet

  • 2020-09-03

ServletContext

ServletContext用于获取上下文对象,关联多个Servlet,是多个Servlet之间 通信的桥梁,用于多个Servlet之间信息的共享。在jsp中我们称其为application。

创建与销毁

tomcat 在启动时, 会为每一个项目创建且仅创建一个ServletContext对象!在服务器关闭或应用被卸载时, 对象会被销毁 !

获取ServletContext

ServletContext context = getServletContext();

web.xml配置

ServletContext方法

  1. 设置&替换数据

    context.setAttribute(String key,Object value);

  2. 获取数据

    Object value = context.getAttribute(String key);

  3. 删除数据

    context.removeAttribute(String key);

  4. 获取项目运行时所在的文件路径

    String 路径 = context.getRealPath("/");

ServletConfig

一个Java类 , 每一个Servlet都拥有一个ServletConfig对象

获取ServletConfig
  1. 在Servlet中,通过getServletConfig()获取!
  2. 通过重写init方法, 使用参数列表中的config对象!

web.xml配置

ServletConfig方法

1.通过ServletConfig如何获取初始化的参数数据
String value = config.getInitParameter(String key);

2.获取所有的key
String[] arr = context.getinitParameterNames();

Servlet 创建时机

web.xml中 serlvet的配置中。
可以通过节点,来指定Servlet启动顺序。
当节点值为负数 或 不写时,表示当用户第一次请求时, 创建此Servlet!
当节点值等于0或大于0时,在Tomcat启动时, 创建此Servlet!
如果存在多个等于0或大于0的Servlet , 启动的顺序是: 值越小越早创建!
值相同时, 自上而下顺序创建!
